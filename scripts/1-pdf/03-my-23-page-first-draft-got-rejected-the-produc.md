**Body:**
I built a cold-run gate: an AI that runs each lesson as a complete beginner before any human sees it. My 23-page draft hit it and stalled. Not because the content was wrong — because length is where beginners quit.

So I didn't trim it. I recompiled it against 11 authoring principles I'd mined from real failures. P1: terminal-first, every step a paste-able command, never "open the editor and find line 14." P3: always coaching, never just doing the work for them. P4: build the two-pane habit so muscle memory forms. The output of those rules isn't a shorter essay. It's a different artifact: one page, one prompt, a beginner pastes it and finishes.

Twenty-three pages became one. Then it became thirteen — M1 through M13 — each a single-page prompt-pack, each cold-tested so a learner with zero terminal experience can't get stuck.

The lesson for any operator: the rules are the product. The page is just where they print. When the constraints are explicit and the gate is automated, "make it shorter" stops being an opinion and becomes a build step.
