## 💼 LINKEDIN CUT
**Hook line:** My game studio has nine employees. None of them are human, and none of them have ever spoken to each other.

**Body:**
I open Claude Code, I type one word — `go` — and a swarm of nine agents wakes up. Builder implements the issue. QA runs a Playwright smoke test on every PR. Product reads the player data. Monetization tunes ad placement. Content fills the pipeline. Competitor scans the market. The README badge row says it plainly: nine agents, all active.

Here's the part most people get wrong when they try to build "multi-agent systems." They reach for a shared database, an orchestration layer, inter-agent messaging. Mine has none of that. The agents coordinate through exactly one mechanism: a label on a GitHub issue. One agent drops a `build-request`. The next picks it up, flips it to `building`, then `done`. There is no shared state to corrupt. The system scales by addition — a new agent is one folder and one registry entry. Remove one, remove one folder.

The receipts live in a file called runs.jsonl. It's at 159 logged runs. Every dispatch, every build, every review — appended, never narrated. If it isn't in that file, it didn't happen.

And my job inside all of it? I review. The swarm proposes, I dispose. I approve the green PR. That's the whole point — I'm not building a game, I built the thing that builds games. The leverage isn't that I work harder. It's that I built the exoskeleton.
